## 🏗️ Overview
SE2-project is a Software Engineering 2 design project centred on a platform for student programming challenges, submissions, automated testing, and score tracking.

The repository is documentation-driven rather than implementation-driven. Its main deliverables are the requirements document (`RASDv1.pdf`), the design document (`DDv1.pdf`), UML models across multiple views, and Alloy specifications for formal analysis.

On GitHub, the repository is archived and read-only. For a portfolio, it is best understood as evidence of requirements engineering, architecture design, and system-level reasoning rather than as a shipped software product.

## 🛠️ Technologies
- Requirements analysis and specification
- Software architecture design
- UML modeling
- Alloy formal modeling
