1. Brain Stroke Prediction with Machine Learning Techniques
Role: Co-author / project contributor
Context: Reproduction and methodological evaluation project on stroke risk prediction

Project overview:
This project aimed to replicate and critically evaluate the paper “Stroke Risk Prediction with Machine Learning Techniques” by Dritsas and Trigka. The work did not stop at straightforward replication. It explicitly compared a paper-faithful pipeline against a more rigorous best-practices pipeline, with emphasis on evaluation validity, class imbalance handling, and the effect of data leakage on reported performance.

Methods and workflow:
- Exploratory data analysis
- Data cleaning and preprocessing
- Missing-value handling
- Outlier analysis and treatment
- One-hot encoding for categorical variables
- Normalization of numerical variables
- Class imbalance handling with SMOTE and SMOTENC comparisons
- 80/20 train-test split
- Hyperparameter tuning with GridSearchCV and 10-fold cross-validation

Data handling details described in the report:
- In notebook A, the team closely replicated the original paper, including oversampling the full dataset.
- In notebook B, the team oversampled only the training set, preserving the original test distribution for fairer evaluation.
- Missing BMI values were dropped in notebook A but imputed in notebook B to reduce information loss.
- Unknown smoking-status labels were reclassified in notebook B using a Random Forest model trained without the stroke label to reduce bias.
- Outliers were handled differently across the two notebooks, including IQR-based treatment in notebook B.

Models used:
- Logistic Regression
- Decision Tree Classifier
- Random Forest Classifier
- Stochastic Gradient Descent Classifier
- k-Nearest Neighbors
- Gaussian Naive Bayes
- Multi-Layer Perceptron
- Stacking Classifier
- Majority Voting Classifier

Feature analysis:
The report states that feature ranking through Random Forest, information gain, and mutual information consistently identified age as the most influential feature, with BMI and average glucose level also ranking highly.

Results and conclusions:
- In notebook A, where the full dataset was oversampled, the best models were stacking, random forest, and majority voting with ROC AUC values of 96%, 94%, and 93%.
- In notebook B, evaluation on a more realistic untouched test set led to substantially lower precision and recall, highlighting that the notebook A setup produced optimistic results.
- Under the fairer evaluation setup, models such as k-NN and logistic regression emerged as stronger candidates on unseen data.
- The project’s main methodological contribution was showing how resampling strategy and test-set contamination can materially alter conclusions in medical ML.

Why this project is notable:
This project shows strong attention to reproducibility, evaluation rigor, and methodological correctness, rather than only maximizing benchmark metrics.

2. Lung-Cancer-Classification
Role: Repository author / contributor
Context: Deep learning project for lung cancer malignancy classification from CT scans

Project overview:
This project, presented in the repository as “DeepBreath AI,” studies lung cancer malignancy classification from CT slices. It evaluates both binary and five-class formulations of the task and compares full-slice versus zoomed-slice representations centered on the nodule.

Dataset and tasks:
- CT data from 2,363 patients in NRRD format with Hounsfield Unit values
- Binary classification: classes 1–3 grouped as benign, classes 4–5 as malignant
- Multi-class classification: five malignancy classes
- Two image regimes: full slice and zoomed-in slice

Preprocessing and training pipeline:
- Min-max normalization
- Hounsfield Unit windowing with width 1300 and level -220
- Data augmentation
- Random oversampling for binary classification
- SMOTE plus class weighting for multi-class classification
- EfficientNetV2S for binary full-slice and binary zoomed-slice tasks
- EfficientNetB1 for full-slice multi-class classification
- EfficientNetB0 for zoomed-slice multi-class classification
- Focal losses
- Adam optimizer
- EarlyStopping and ReduceLROnPlateau callbacks

Reported results from the repository README:
Full-slice binary classification:
- Accuracy: 0.73 ± 0.02
- Precision: 0.44 ± 0.03
- Recall: 0.52 ± 0.05
- F1-score: 0.48 ± 0.03
- AUC: 0.66 ± 0.02

Full-slice multi-class classification:
- Accuracy: 0.23 ± 0.04
- Precision: 0.21 ± 0.04
- Recall: 0.24 ± 0.04
- F1-score: 0.19 ± 0.04
- AUC: 0.51 ± 0.04

Zoomed-slice binary classification:
- Accuracy: 0.72 ± 0.03
- Precision: 0.45 ± 0.05
- Recall: 0.63 ± 0.08
- F1-score: 0.52 ± 0.05
- AUC: 0.75 ± 0.04

Zoomed-slice multi-class classification:
- Accuracy: 0.53 ± 0.02
- Precision: 0.49 ± 0.03
- Recall: 0.56 ± 0.02
- F1-score: 0.47 ± 0.03
- AUC: 0.74 ± 0.02

Key findings:
- Binary classification was more reliable than five-class classification.
